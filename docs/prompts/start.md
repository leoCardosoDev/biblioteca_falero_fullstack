## Start Session — AI Guardrail Initialization

> **Before any analysis, planning, or implementation:**
>
> 1. Read and internalize `app/docs/ai/guardrail.prompt.md`
> 2. Confirm you understand the pre-implementation checklist
> 3. Identify which modules will be affected by this work
> 4. Read the relevant `app/docs/modules/*.module.md` for affected modules
> 5. Read the relevant `app/docs/domain/*.rules.md` for affected aggregates
>
> **Acknowledge that you have read and will follow:**
>
> * `.agent/standards/STANDARD_GENERAL.md`
> * `.agent/standards/STANDARD_BACKEND.md` (if backend work)
> * `.agent/standards/STANDARD_FRONTEND.md` (if frontend work)
> * `.agent/standards/STANDARD_GITFLOW.md`
>
> **Only proceed when guardrail is acknowledged.**
>
> Expected response format:
> ```
> ✅ Guardrail loaded
> ✅ Affected modules: [list]
> ✅ Module contracts read: [list]
> ✅ Domain rules read: [list]
> ✅ Ready to proceed
> ```
